## Overview

Foundational ULID-keyed lookup tables backing the rest of the schema. These tables provide the taxonomy used by campaigns, entries, creator profiles, and social account syncing. Bundled with feature 1.9 (lookup seeders) because the seed data is small and the two features have no value in isolation.

## What shipped

### Migrations
- `platforms` — social platforms (TikTok, Instagram, YouTube, etc.)
- `niches` — creator content niches (Fitness, Beauty, Tech, etc.)
- `industries` — brand industries (Health & Wellness, Fashion, etc.)
- `content_types` — video formats (GRWM, Unboxing, Review, etc.)

All four use identical shape: `ulid` PK, unique `name`/`slug`, optional `icon_url` (platforms/niches only), `description` (content_types only), `is_active` flag, `sort_order`, timestamps.

### Models
- `App\Models\Platform`
- `App\Models\Niche`
- `App\Models\Industry`
- `App\Models\ContentType`

All `final`, use `HasUlids`, with `is_active` → bool and `sort_order` → int casts.

### Seeders
- `PlatformSeeder` — 6 platforms
- `NicheSeeder` — 20 niches
- `IndustrySeeder` — 16 industries
- `ContentTypeSeeder` — 10 content types

All seeders use `updateOrCreate` keyed on `slug` → idempotent and safe to re-run. Wired into `DatabaseSeeder::run()`.

## Design decisions

- **ULID PKs** per schema spec — safe for URLs, chronologically sortable.
- **Lookup tables, not enums** — values grow over time, need labels/icons/ordering, and admin UI will manage them.
- **`sort_order`** — UI controls display order without alphabetical sorting.
- **`is_active`** — soft-disable without deletion to preserve FK integrity.
