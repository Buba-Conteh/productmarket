# Feature 1.8 — Messaging, Growth & Analytics

**Branch:** `feature/1.8-messaging-growth-analytics`
**Roadmap ref:** Phase 1 · Feature 1.8 — Database migrations — messaging, growth, analytics
**Status:** 🟢 Complete

## Overview

Adds the remaining database tables that complete the full schema spec: per-entry message threads, referral tracking, and daily/weekly analytics snapshot tables. After this feature, all 60+ tables from `context/database-schema.md` exist in the database.

## Migrations

| File | Purpose |
|---|---|
| `2026_04_16_100000_create_message_threads_table.php` | One thread per entry (unique FK), `last_message_at` for sorting |
| `2026_04_16_100001_create_messages_table.php` | Body + read_at, FK to thread and sender user, indexed on (thread_id, created_at) |
| `2026_04_16_100002_create_referrals_table.php` | Referrer/referred user FKs, unique referral_code, type enum (creator/brand), status state machine |
| `2026_04_16_100003_create_campaign_analytics_table.php` | Daily snapshots — entries, live count, views, paid out, top entry. Unique on (campaign_id, date) |
| `2026_04_16_100004_create_creator_analytics_table.php` | Weekly snapshots — views, earned, entries count, avg engagement. Unique on (creator_profile_id, week_start) |

## Models

- `MessageThread` — belongsTo Entry, hasMany Messages. Datetime cast on last_message_at.
- `Message` — belongsTo MessageThread (thread_id), belongsTo User (sender_user_id). Datetime cast on read_at.
- `Referral` — belongsTo User as referrer and referred. Datetime casts on qualified_at and rewarded_at.
- `CampaignAnalytic` — belongsTo Campaign, belongsTo Entry (top_entry_id). No timestamps (snapshot table).
- `CreatorAnalytic` — belongsTo CreatorProfile. No timestamps (snapshot table).

## Inverse relations added

- `User::referralsMade()` and `User::referralsReceived()`
- `Entry::messageThread()` (HasOne)
- `Campaign::analytics()` (HasMany CampaignAnalytic)
- `CreatorProfile::analytics()` (HasMany CreatorAnalytic)
