# Feature 6.5 — Ripple Milestone Payout

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

Every time `SyncEntryViewCountJob` runs and a Ripple creator crosses a milestone threshold, a payout is created and dispatched automatically. Budget cap and per-creator cap are both enforced before each milestone fires.

---

## Changes

### `app/Services/Social/ViewSyncService.php`

- Injected `PayoutService` via constructor
- `detectAndQueueMilestones()` return type changed `void → array<string>` (payout IDs)
- Inline `Payout::create()` + fee math replaced with `PayoutService::createPayout()`
- `syncEntryPlatform()` collects returned payout IDs via reference, then dispatches `ProcessPayoutJob` for each after the transaction commits

### `app/Jobs/ResolveContestDeadlineJob.php`

- Replaced private `createPayout()` helper (inline fee math + `Payout::create()`) with `PayoutService::createPayout()`
- Collected created payouts from DB transaction, dispatches `ProcessPayoutJob` after commit
- Also covers the deadline-triggered contest prize path (completes 6.3 for the automated resolution case)

---

## Flow

```
SyncAllLiveEntriesJob (every 6h)
  → SyncEntryViewCountJob per entry
  → ViewSyncService::syncEntry()
  → syncEntryPlatform()
  → DB transaction:
      pivot.verified_view_count updated
      detectAndQueueMilestones(previousCount, newCount)
        → for each crossed milestone:
            budget cap check
            per-creator cap check
            PayoutService::createPayout(entry, 'ripple_milestone', gross)
              → Payout created (status: pending)
              → creator.pending_earnings += net
            EntryRippleEarning created (type: milestone)
            ripple_details.budget_spent += gross
        → returns [payoutId, ...]
  → ProcessPayoutJob::dispatch(payoutId) for each
     ↓ (async)
  → PayoutService::executeTransfer()
  → Stripe transfer to creator
  → payout.status = paid | failed
```
