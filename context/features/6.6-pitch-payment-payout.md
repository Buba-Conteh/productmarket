# Feature 6.6 — Pitch Payment Payout

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

When a brand confirms that a creator's Pitch entry post is live, the accepted bid amount is released to the creator. Platform takes 15%, creator receives 85%.

---

## Root Cause Fixed

`EntryService::confirmPitchLive()` was calling `payoutService->createPayout()` but discarding the return value. `ProcessPayoutJob` was never dispatched, leaving payout records in `pending` status indefinitely.

---

## Changes

### `app/Services/EntryService.php` — `confirmPitchLive()`

- Capture payout by reference inside the DB transaction
- Dispatch `ProcessPayoutJob` after transaction commits (same pattern as `approveRipple()` and `selectContestWinner()`)

---

## Flow

```
Brand clicks "Confirm Post is Live"
  → POST /campaigns/{campaign}/entries/{entry}/confirm-live
  → BrandEntryController::confirmPitchLive()
  → EntryService::confirmPitchLive()
  → DB transaction:
      PayoutService::createPayout(entry, 'pitch_payment', accepted_bid)
        → Payout created (status: pending)
        → creator.pending_earnings += net_amount
  → ProcessPayoutJob::dispatch(payout->id)  ← was missing, now fixed
     ↓ (async, queued)
  → PayoutService::executeTransfer()
  → Stripe Connect transfer to creator (net_amount)
  → payout.status = paid | failed
  → on paid: creator.total_earned += net, pending_earnings -= net
             escrow.total_released += gross
```
