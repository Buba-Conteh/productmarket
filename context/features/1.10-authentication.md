# Feature 1.10 — User Authentication

**Branch:** `feature/1.9-1.10-seeders-auth`
**Roadmap ref:** Phase 1 · Feature 1.10 — User authentication
**Status:** 🟢 Complete

## Overview

Full authentication system powered by Laravel Fortify with Inertia/React frontend pages. Covers registration, login, logout, email verification, password reset, password confirmation, and two-factor authentication.

## What was already scaffolded

The Laravel starter kit + Fortify provided:

- **FortifyServiceProvider** — configures all Inertia view bindings, rate limiting, and action classes
- **CreateNewUser** action — validates and creates user on registration
- **ResetUserPassword** action — validates and resets forgotten password
- **Fortify config** — enables registration, password resets, email verification, and 2FA with confirmation

## Changes in this feature

- **User model** now `implements MustVerifyEmail` — ensures Fortify's email verification feature actually enforces the contract. Without this, the `verified` middleware would not block unverified users.

## Auth pages (React/Inertia)

| Page | Route | Purpose |
|---|---|---|
| `auth/register.tsx` | `/register` | Name, email, password registration form |
| `auth/login.tsx` | `/login` | Email + password login with remember me |
| `auth/forgot-password.tsx` | `/forgot-password` | Request password reset link |
| `auth/reset-password.tsx` | `/reset-password/{token}` | Set new password from reset email |
| `auth/verify-email.tsx` | `/email/verify` | Prompt to verify email address |
| `auth/confirm-password.tsx` | `/user/confirm-password` | Re-enter password for sensitive actions |
| `auth/two-factor-challenge.tsx` | `/two-factor-challenge` | 2FA code entry during login |

## Fortify features enabled

```php
Features::registration()
Features::resetPasswords()
Features::emailVerification()
Features::twoFactorAuthentication(['confirm' => true, 'confirmPassword' => true])
```
