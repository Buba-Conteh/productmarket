# Feature 6.4 — Ripple Initial Fee Payout

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

When a brand approves a Ripple entry, the creator receives a flat upfront fee immediately. The payout is created inside a DB transaction, then `ProcessPayoutJob` is dispatched after commit to execute the Stripe Connect transfer.

---

## Change

Single method updated: `EntryService::approveRipple()` in [app/Services/EntryService.php](../../app/Services/EntryService.php).

**Before:** payout record created inline, no transfer dispatched.  
**After:** payout captured from transaction via reference, `ProcessPayoutJob::dispatch($payout->id)` fired after commit.

---

## Flow

```
Brand clicks "Approve" on a Ripple entry
  → BrandEntryController::approveRipple()
  → EntryService::approveRipple()
  → DB transaction:
      entry.status = approved
      PayoutService::createPayout(entry, 'ripple_initial_fee', initial_fee)
        → Payout record created (status: pending)
        → creator.pending_earnings += net
      EntryRippleEarning created (type: initial_fee, milestone_number: 0)
      campaign_ripple_details.budget_spent += gross
  → ProcessPayoutJob::dispatch(payout.id)
     ↓ (async)
  → PayoutService::executeTransfer()
  → Stripe transfer to creator's Connect account
  → payout.status = paid | failed
  → creator.total_earned += net (on success)
  → escrow.total_released += gross (on success)
```
