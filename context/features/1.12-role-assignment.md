# Feature 1.12 — Role Assignment on Registration

**Branch:** `feature/1.11-1.12-social-login-roles`
**Roadmap ref:** Phase 1 · Feature 1.12 — Role assignment on registration
**Status:** 🟢 Complete

## Overview

Spatie Permission manages three roles — `brand`, `creator`, `admin`. Roles are assigned at registration time based on the user's selection. The admin role is not available via self-registration.

## Implementation

### Packages

- `spatie/laravel-permission` — role and permission management

### Migration

Published Spatie's `create_permission_tables` migration with one modification: `model_morph_key` columns changed from `unsignedBigInteger` to `string(26)` to match ULID primary keys on the User model.

### Seeder

`RoleSeeder` creates three roles (brand, creator, admin) using `Role::findOrCreate()` for idempotency. Wired into `DatabaseSeeder`.

### User model

- Added `HasRoles` trait from Spatie Permission
- Added `MustVerifyEmail` interface (from 1.10)

### Registration flow

`CreateNewUser` action updated:
- Validates `role` field — required, must be `brand` or `creator` (admin not allowed)
- Wraps user creation + role assignment in a DB transaction
- `$user->assignRole($input['role'])` called after creation

### Frontend

Register page updated with a "I am a" select dropdown (Brand / Creator) between email and password fields.

### Tests

4 registration tests (up from 1):
- Register as brand — verifies role is assigned
- Register as creator — verifies role is assigned
- Rejects invalid role (e.g. `admin`)
- Fails without a role
