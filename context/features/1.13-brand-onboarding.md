# Feature 1.13 — Brand Onboarding Flow

**Branch:** `feature/1.13-1.14-onboarding`
**Roadmap ref:** Phase 1 · Feature 1.13 — Brand onboarding flow
**Status:** 🟢 Complete

## Overview

3-step onboarding wizard for brands after registration. Users who register with the `brand` role are redirected to the onboarding flow before they can access the dashboard or any protected route.

## Implementation

### Migration

`2026_04_16_150000_add_onboarding_completed_at_to_profiles` adds `onboarding_completed_at` timestamp (nullable) to both `brand_profiles` and `creator_profiles` tables. This field tracks whether a user has completed onboarding.

### Middleware

`EnsureOnboardingComplete` — checks if the authenticated user has the `brand` or `creator` role and whether their profile exists with `onboarding_completed_at` set. If not, redirects to the first step of their respective onboarding flow. Applied to dashboard and settings routes.

### Spatie Permission Middleware

Registered `role`, `permission`, and `role_or_permission` middleware aliases in `bootstrap/app.php` for Spatie Permission. Onboarding routes are protected by `role:brand` and `role:creator` respectively.

### Controller

`App\Http\Controllers\Onboarding\BrandOnboardingController` — handles the 3-step flow:

1. **Company profile** (`GET/POST /onboarding/brand/company`) — company name (required), industry (required), website (optional), description (optional). Creates or updates `BrandProfile` via `updateOrCreate`.
2. **Billing** (`GET/POST /onboarding/brand/billing`) — placeholder step for Stripe Cashier integration (Phase 2). Currently skippable.
3. **Tour** (`GET /onboarding/brand/tour`, `POST /onboarding/brand/complete`) — dashboard feature tour showing what brands can do on the platform. Completing this step sets `onboarding_completed_at` and redirects to dashboard.

Each step checks if the user is already onboarded and redirects to dashboard. Steps 2 and 3 redirect to step 1 if no profile exists yet.

### Routes

All routes under `onboarding/brand/*` prefix, protected by `auth`, `verified`, and `role:brand` middleware. Defined in `routes/onboarding.php`.

### Frontend

- **Onboarding layout** — clean wizard-style layout with logo header, step indicator with numbered circles and connecting lines, centered content area. No sidebar.
- **Company page** — form with company name, industry dropdown (populated from `industries` table), website URL, and description textarea.
- **Billing page** — informational card showing available plans ($49/$149/$399) with "Continue — set up billing later" button.
- **Tour page** — feature cards (Launch campaigns, Discover creators, Secure payments, Track performance) with "Go to dashboard" button.
