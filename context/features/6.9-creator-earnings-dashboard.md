# Feature 6.9 — Creator Earnings Dashboard

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

Creators can view their total earned, pending earnings, and full paginated payout history at `/wallet` (already in the sidebar nav).

---

## Changes

### `app/Http/Controllers/Creator/EarningsController.php` (new)
- `index()` loads creator profile, paginates payouts with campaign eager load
- Passes: `total_earned`, `pending_earnings`, `paid_count`, `stripe_connect_status`, `payouts` (paginated)

### `routes/creator.php`
- Added `GET /wallet` → `EarningsController::index()` → `creator.earnings.index`

### `resources/js/pages/creator/earnings.tsx` (new)
- 3 stat cards: Total Earned (primary), Pending Payouts, Paid Payouts count
- Stripe account status warning banner when not active
- Paginated table: Campaign, Type, Gross, Fee, Net, Status (badge), Date
- Status badges with icons: paid (green), pending (yellow), processing (blue), failed (red)
- Failure reason shown inline under failed status
- Empty state card when no payouts exist

---

## Route

```
GET /wallet  →  creator.earnings.index  (auth + verified + role:creator)
```
