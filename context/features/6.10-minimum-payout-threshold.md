# Feature 6.10 — Minimum Payout Threshold

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

Payout transfers are held until a creator's accumulated `pending_earnings` meets or exceeds `platform_settings.min_creator_payout`. Once the threshold is reached an hourly job re-dispatches all held pending payouts for eligible creators.

---

## Changes

### `app/Services/PayoutService.php` — `executeTransfer()`
- Loads `min_creator_payout` from `PlatformSetting::current()`
- If min > 0 and `creator.pending_earnings < min`: returns early, payout stays `pending` (no failure, no retry consumed)

### `app/Jobs/ReleaseHeldPayoutsJob.php` (new)
- Loads `min_creator_payout`; exits immediately if min is 0 (threshold disabled)
- Chunks through all `pending` payouts where the creator's `pending_earnings >= min`
- Dispatches `ProcessPayoutJob` for each — they will now pass the threshold check and execute

### `routes/console.php`
- Schedules `ReleaseHeldPayoutsJob` hourly with overlap protection

---

## Flow

```
ProcessPayoutJob::handle()
  → PayoutService::executeTransfer()
      if pending_earnings < min_creator_payout → return (stay pending)
      else → Stripe transfer proceeds

Hourly: ReleaseHeldPayoutsJob
  → find all pending payouts where creator.pending_earnings >= min
  → ProcessPayoutJob::dispatch(payoutId) for each
  → jobs run → threshold now met → transfers execute
```
