# Feature 6.1 — Stripe Connect Creator Onboarding

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

Stripe Connect Express onboarding is embedded in step 4 (Payout) of the creator onboarding flow. Creators connect a bank account or debit card through Stripe's hosted onboarding UI. KYC is handled entirely by Stripe.

---

## Implementation

### Service — `app/Services/StripeConnectService.php`

Wraps all Stripe Connect API calls:

- `createExpressAccount(CreatorProfile)` — creates a Stripe Express account with `transfers` capability, stores `stripe_connect_id` and sets status to `pending`
- `createOnboardingLink(CreatorProfile, returnUrl, refreshUrl)` — generates a time-limited `account_onboarding` link
- `syncAccountStatus(CreatorProfile)` — fetches the account from Stripe and resolves status to `pending | restricted | active` based on `details_submitted` and `charges_enabled`

### Controller — `app/Http/Controllers/Onboarding/CreatorOnboardingController.php`

Four new actions added:

| Method | Route | Purpose |
|---|---|---|
| `connectStripe` | `POST /onboarding/creator/stripe/connect` | Create account (if needed) + redirect to Stripe |
| `stripeReturn` | `GET /onboarding/creator/stripe/return` | Sync status, redirect back to payout step |
| `stripeRefresh` | `GET /onboarding/creator/stripe/refresh` | Regenerate expired link, redirect back to Stripe |

The `payout()` method now passes `stripe_connect_id` and `stripe_connect_status` to the frontend.

### Routes — `routes/onboarding.php`

Three new routes added inside the `role:creator` group.

### Frontend — `resources/js/pages/onboarding/creator/payout.tsx`

The payout page now contains two sections:

1. **`StripeConnectSection`** component — renders one of three states:
   - **Not connected**: "Set up payouts with Stripe" button → POST to connect route
   - **Pending/incomplete**: spinner + "Continue Stripe setup" button
   - **Active**: green checkmark "Payouts connected"

2. **Plan selection** (existing Free / Creator Pro cards) — unchanged

Stripe setup is optional during onboarding; creators can complete it later from Settings.

---

## Data Flow

```
Creator clicks "Set up payouts"
  → POST /onboarding/creator/stripe/connect
  → StripeConnectService::createExpressAccount() (if no stripe_connect_id)
  → StripeConnectService::createOnboardingLink()
  → redirect()->away(stripeUrl)

Stripe onboarding complete
  → GET /onboarding/creator/stripe/return
  → StripeConnectService::syncAccountStatus()
  → redirect to payout step (status now active/restricted)

Link expired
  → GET /onboarding/creator/stripe/refresh
  → StripeConnectService::createOnboardingLink() (fresh link)
  → redirect()->away(newStripeUrl)
```

## Dependencies

- `STRIPE_KEY` and `STRIPE_SECRET` must be set in `.env`
- Laravel Cashier is already installed (`config/cashier.php` uses `CASHIER_SECRET` → mapped to `STRIPE_SECRET`)
- `creator_profiles.stripe_connect_id` and `stripe_connect_status` columns already exist from Phase 1 migrations
