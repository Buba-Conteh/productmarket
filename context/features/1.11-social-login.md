# Feature 1.11 — Social Login

**Branch:** `feature/1.11-1.12-social-login-roles`
**Roadmap ref:** Phase 1 · Feature 1.11 — Social login
**Status:** 🟢 Complete

## Overview

Social authentication via Laravel Socialite. Google is available for both brands and creators. LinkedIn is available for brands. TikTok social login will be added when TikTok OAuth is implemented for view tracking in Phase 5.

## Implementation

### Packages

- `laravel/socialite` — handles OAuth redirect/callback flows

### Config

`config/services.php` updated with:
- `google` — client_id, client_secret, redirect via env vars
- `linkedin-openid` — client_id, client_secret, redirect via env vars

### Controller

`App\Http\Controllers\Auth\SocialAuthController` — handles redirect and callback for all providers:
- `redirect(provider)` — redirects to provider OAuth consent screen
- `callback(provider)` — handles OAuth callback, finds or creates user, logs in
- Provider validation via allowlist (google, linkedin)
- New users created via social login get `email_verified_at` set immediately (provider already verified the email)

### Routes

```
GET /auth/{provider}/redirect  → social.redirect
GET /auth/{provider}/callback  → social.callback
```

Both routes are behind `guest` middleware.

### Frontend

- Login page updated with "Continue with Google" button above the email/password form
- Visual separator ("Or continue with") between social and email login

### Environment variables required

```
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
LINKEDIN_CLIENT_ID=
LINKEDIN_CLIENT_SECRET=
```

## Notes

- Social login users who don't yet have a role will need to select one on first login (handled in onboarding flow, 1.13/1.14)
- TikTok social login deferred to Phase 5 when TikTok OAuth is needed for view tracking anyway
