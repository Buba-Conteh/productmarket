# Feature 1.15 — Admin Panel Scaffold

**Branch:** `feature/1.13-1.14-onboarding`
**Roadmap ref:** Phase 1 · Feature 1.15 — Admin panel scaffold
**Status:** 🟢 Complete

## Overview

Separate Inertia route group (`/admin`) protected by `auth`, `verified`, and `role:admin` middleware. Provides user management (list, search/filter, status updates) and platform settings (fees, payout splits, referral bonuses). Built with the same shadcn/ui components and design language as the main app, with a dedicated admin sidebar.

## Implementation

### Routes

`routes/admin.php` — all routes under `/admin` prefix, named `admin.*`, protected by `auth`, `verified`, and `role:admin` middleware:

- `GET  /admin`                        → `AdminController@dashboard`      (`admin.dashboard`)
- `GET  /admin/users`                  → `AdminUserController@index`       (`admin.users.index`)
- `GET  /admin/users/{user}`           → `AdminUserController@show`        (`admin.users.show`)
- `PATCH /admin/users/{user}/status`   → `AdminUserController@updateStatus` (`admin.users.update-status`)
- `GET  /admin/settings`              → `AdminSettingsController@edit`    (`admin.settings.edit`)
- `PUT  /admin/settings`              → `AdminSettingsController@update`  (`admin.settings.update`)

Registered via `require __DIR__.'/admin.php'` in `routes/web.php`.

### Controllers

**`App\Http\Controllers\Admin\AdminController`**
- `dashboard()` — returns stats: total users, brands, creators, suspended/banned counts, new users today and this week.

**`App\Http\Controllers\Admin\AdminUserController`**
- `index()` — paginated user list (20/page) with search (name/email), role filter (brand/creator/admin), status filter (active/suspended/banned). Eager-loads roles. Passes `users` paginator + `filters` to frontend.
- `show()` — single user detail page with roles, brandProfile, creatorProfile, social accounts.
- `updateStatus()` — validates status enum (`active`, `suspended`, `banned`); blocks admin from changing their own status; updates `users.status`.

**`App\Http\Controllers\Admin\AdminSettingsController`**
- `edit()` — returns current `PlatformSetting` (row id=1).
- `update()` — validates all fields, checks contest split sum ≤ 100%, updates the singleton row.

### Frontend

**Layout** — `resources/js/layouts/admin-layout.tsx` → `layouts/admin/admin-sidebar-layout.tsx`
- Uses `AdminSidebar` component instead of `AppSidebar`
- Same `AppShell`/`AppContent`/`AppSidebarHeader` shell as main app

**`AdminSidebar`** (`components/admin-sidebar.tsx`)
- Grouped nav: Overview (Dashboard), Manage (Users), Platform (Settings)
- Orange "Admin Panel" badge under the logo to visually distinguish from the main app
- Collapses to icons same as `AppSidebar`

**Pages**

| Page | Path | Description |
|---|---|---|
| Dashboard | `pages/admin/dashboard.tsx` | Stats cards (totals, role split, moderation counts) + quick-link cards |
| Users index | `pages/admin/users/index.tsx` | Search input, role/status filters, paginated table, inline status actions via dropdown |
| Settings | `pages/admin/settings.tsx` | Cards for platform fee, contest splits (with live total display), payout threshold, referral bonuses |

Each page uses `ComponentName.layout = (page) => <AdminLayout ...>{page}</AdminLayout>` for the Inertia persistent layout pattern.

### Design Details

- Orange accent (`text-orange-500`, `bg-orange-500/10`) matches the app's brand color seen in design screenshots
- Left border accent on stat cards differentiates metric categories
- Contest split total turns green at 100%, red if exceeded — live validation feedback
- Actions dropdown per user row (Activate / Suspend / Ban) with disabled state for current status and "You" label for the admin's own row
