# Feature 6.8 — Escrow Refund on Campaign Cancellation

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

When a brand cancels a campaign, the unspent escrow budget (`total_held - total_released`) is refunded via Stripe. For Ripple campaigns this covers any budget not yet paid to creators; for Contest/Pitch it's the full escrow if no payouts were made.

---

## Changes

### Migration: `add_refund_fields_to_escrow_transactions_table`
- `stripe_refund_id` string nullable — Stripe refund object ID (`re_xxx`)
- `refunded_at` timestamp nullable — when the refund was issued

### `app/Models/EscrowTransaction.php`
- Added `stripe_refund_id` and `refunded_at` to `$fillable` and `casts`

### `app/Services/CampaignService.php`
- Added `StripeClient` constructor injection (same pattern as `PayoutService`)
- `cancel()` now calls `issueStripeRefund()` and stores the refund ID + timestamp
- New private `issueStripeRefund()` method:
  - Skips if PI ID is a placeholder (`pending_` prefix — pre-Stripe-integration campaigns)
  - Issues a partial Stripe refund for the unspent amount in cents
  - Logs and returns `null` on Stripe error (DB still marked refunded for admin review)

---

## Flow

```
Brand cancels campaign
  → CampaignService::cancel()
  → DB transaction:
      refundable = total_held - total_released
      issueStripeRefund(escrow, refundable)
        → if PI is placeholder: skip, return null
        → stripe->refunds->create(payment_intent, amount_cents)
        → on success: return refund.id
        → on failure: Log::error, return null
      escrow.update(total_refunded, stripe_refund_id, status=refunded, refunded_at)
      campaign.status = cancelled
```
