## Overview

Schema for the user account layer: the core `users` table migrated to ULID with platform-specific columns, plus the role-scoped profile tables (`brand_profiles`, `creator_profiles`), the creator↔niche pivot, encrypted social account linkage, and agency team membership. This is the identity foundation every other feature hangs off.

## What shipped

### Users table changes
Modified the original `create_users_table` migration in place (no prod data yet):
- PK changed from auto-increment `id()` → `ulid('id')->primary()`
- `sessions.user_id` changed from `foreignId` → `foreignUlid`
- Added `avatar` (R2 path), `country` (ISO-2), `status` enum (`active`/`suspended`/`banned`), `last_active_at`

### New tables
| Table | Purpose |
|---|---|
| `brand_profiles` | 1:1 with brand users — company_name, website, industry FK, logo, stripe_customer_id, is_agency |
| `creator_profiles` | 1:1 with creator users — display_name, bio, stripe_connect_id, stripe_connect_status enum, denormalized total_earned / pending_earnings |
| `creator_niches` | Composite-PK pivot linking creator_profiles ↔ niches |
| `social_accounts` | OAuth-linked platform accounts with encrypted tokens, follower/engagement metrics, verified flag; unique on (user_id, platform_id) |
| `agency_members` | Team members under an agency brand — role enum (owner/manager/viewer), invited_at/accepted_at |

### Models
- `User` — added `HasUlids`, plus `brandProfile()`, `creatorProfile()`, `socialAccounts()` relations, `last_active_at` datetime cast
- `BrandProfile`, `CreatorProfile`, `SocialAccount`, `AgencyMember` — all `final`, `HasUlids`, relationships wired
- `SocialAccount` casts `oauth_token` and `oauth_refresh_token` as `encrypted` and hides them from array output

### Fixes
- `App\Concerns\ProfileValidationRules::profileRules()` and `emailRules()` — widened `$userId` param from `?int` to `int|string|null` to accept ULIDs (previously broke two profile update tests)

## Design decisions

- **Modified the original users migration in place** rather than adding an `alter` migration. Safe because there is no production data; keeps history clean.
- **ULIDs everywhere** per schema spec — safe for public URLs, chronologically sortable.
- **Denormalized earnings on `creator_profiles`** — `total_earned` / `pending_earnings` updated by payout jobs to avoid aggregation queries on the dashboard.
- **Encrypted OAuth tokens** — Laravel's `encrypted` cast on `social_accounts`, never stored plaintext.
- **Unique(user_id, platform_id)** on social_accounts — one account per platform per user.
- **Spatie Permission for roles** — no role column on users; roles assigned via pivot (to be wired in feature 1.12).
