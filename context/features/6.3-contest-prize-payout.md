# Feature 6.3 — Contest Prize Payout

**Phase:** 6 — Payouts  
**Status:** Complete  
**Branch:** feature/1.13-1.14-onboarding

---

## Overview

When a brand selects a contest winner, prize payouts (winner + optional runner-up) are created and dispatched as queued jobs that execute Stripe Connect transfers to the creators' accounts.

---

## Implementation

### `app/Jobs/ProcessPayoutJob.php` *(new — generic, reused by 6.4–6.6)*

Queued job that:
- Looks up the `Payout` by ID
- Guards: skips if payout is not in `pending` status
- Calls `PayoutService::executeTransfer()`
- `tries = 1` — no automatic retries; 6.7 handles retry/alerting

### `app/Services/EntryService.php` *(refactored)*

- Injected `PayoutService` via constructor
- All three inline fee calculations replaced with `PayoutService::createPayout()`:
  - `approveRipple()` — `ripple_initial_fee`
  - `confirmPitchLive()` — `pitch_payment`
  - `selectContestWinner()` — `contest_prize` + `contest_runner_up`
- Removed duplicate `Payout` model import (now handled inside PayoutService)

### Contest winner flow in `selectContestWinner()`

Inside the DB transaction:
1. Mark winning entry status → `won`
2. Set `winner_entry_id` + `winner_selected_at` on `campaign_contest_details`
3. Mark all other `pending_review` entries → `not_selected`
4. `PayoutService::createPayout($entry, 'contest_prize', $prize_amount)` → Payout record + pending_earnings incremented
5. If `runner_up_prize > 0`: find first `not_selected` entry, `createPayout(... 'contest_runner_up', ...)`
6. Campaign status → `completed`

After the transaction: `ProcessPayoutJob::dispatch($payout->id)` for each payout (winner + runner-up).

---

## Data Flow

```
Brand clicks "Select winner"
  → BrandEntryController::selectWinner()
  → EntryService::selectContestWinner()
  → DB transaction: entry=won, others=not_selected, payouts created, campaign=completed
  → ProcessPayoutJob dispatched (winner payout)
  → ProcessPayoutJob dispatched (runner-up payout, if applicable)
     ↓ (async via queue)
  → PayoutService::executeTransfer()
  → Stripe transfers->create() with net_amount in cents
  → payout.status = paid | failed
  → creator.total_earned += net (on success)
  → escrow.total_released += gross (on success)
```
